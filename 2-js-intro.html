What is JavaScript

Making its first appearance in 1995, JavaScript was created by an engineer at Netscape to provide a user-friendly, lightweight programming language that can be easily adapted for the rise of the Internet. Now, with HTML and CSS, it is one of the core languages of the Internet and is growing quickly to accommodate beyond the client-side.

JavaScript is powerful enough to:

  • create interactivity on webpages
  • web based games
  • edit PDF files (such as inserting the date into the document when the file is opened)
  • even manipulate SVG image files with amazing animations

JavaScript allows web pages to do more than just “sit there." You can animate, calculate, etc. - you can do it all! It is a great programming bridge between “design” and “development” that allows for endless creativity.

Common confusion: JavaScript is NOT Java

They are largely different programming languages and should not be confused with one another.

JavaScript Syntax

Syntax?

Syntax is just a fancy reference to the arrangement of the code within the editors.

See the Pen Date by Matthew Leach (@shoutmatt) on CodePen.

Basic syntax of JavaScript:

  • var - defines a variable (an object of some value)
  • ; - terminator, commonly found at the end of a code operation
  • "word" - quotes create strings, which are a form of data
  • function() - performs some action or method
  • {} - block notation, contains code that can be initialized by a function
  • . - dot notation, allows for the chaining of variables and functions

 

Variables

A variable is a symbolic name that is used to represent information.

Variables of your Past

x = 1
y = 2
z = x + y; 

What is z??

Looking at the formula above, you may remember how you used to use letters to store numbers. The problem is that x, y, and z don't mean a whole lot at this moment. Using descriptive words for the variables make this easier.

Variables in Code

men = 10
women = 20
totalPeople = men + women

What is totalPeople??

As you can see, the variables for men, women, and totalPeople are much more useful in knowing what kind of data we are working with.

Functions

Functions are blocks of code that perform tasks for us.

In JavaScript, you follow the general syntax:

  1. Declare,
  2. Define,
  3. Call (invoke).

Syntax:

See the Pen LTC - Basic Function by Matthew Leach (@shoutmatt) on CodePen.

~What is the value produced by this function?

More syntax:

  • Parameters - (a,b) - hypothetically what passes through the function
  • Arguments - real values of the parameters the function affects
  • Block - {...} - the function’s operational code
  • Return command - the output of the function

Recap of Conditional Rules

  • If statements perform an action if the statement is True
  • Else statements perform an action if the statement is False
  • Else if statements perform an action if the first is False but the second is True Is there any other way to do this?

= vs == vs ===

 

= :  assignment operator
It sets variables equal to a specific value.

See the Pen LTC = by Matthew Leach (@shoutmatt) on CodePen.

== :  abstract equality comparison
It compares two items to see if they are of equal value, but it ignores if they are the same exact type of data.

See the Pen LTC == by Matthew Leach (@shoutmatt) on CodePen.

=== :  strict equality comparison
It compares the value & type of the items to see if they are exactly the same. In the case of null vs undefined, null is more specifically typed than undefined, so they are not exactly the same value. It is recommended to use === over == as much as possible.

See the Pen LTC === by Matthew Leach (@shoutmatt) on CodePen.
